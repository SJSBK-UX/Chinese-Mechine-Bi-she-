# 药品检测系统 Android App 开发文档

## 1. 开发目标

将现有网页端药品检测系统改造成 Android 手机 App。App 负责用户登录、图片上传、批量图片检测、视频检测、摄像头检测结果展示、历史记录查询和个人信息管理；后端识别计算、数据存储和文件存储部署在阿里云服务器。

本项目不建议把 Spring Boot、Flask、MySQL 或 YOLO 模型塞进手机端。手机端只保留 UI 和交互逻辑，识别任务交给云服务器处理。

## 3. 推荐技术路线

当前采用“云服务器 + Android WebView 壳”的实现方式：

```text
Vue3 页面
  -> 部署到云服务器 Nginx
  -> Android WebView 加载云端页面
  -> Android Studio 打包 APK
  -> App 通过 HTTPS 调用云服务器接口
```

选择 WebView 云端壳的原因：

- 复用现有 Vue 页面，改造成本低。
- 前端更新后只需要重新部署服务器，手机端通常不用重新发 APK。
- 对 4 核 8G 云服务器方案最直接，YOLO 推理、数据库和文件都留在云端。
- 后续如果要把静态资源内置到 APK，再迁移到 Capacitor 也不影响当前后端架构。

## 4. 云服务器接口规划

建议准备一个域名，例如：

```text
https://medicine.example.com
```

Nginx 对外统一暴露路径：

```text
https://medicine.example.com/api/     -> Spring Boot 127.0.0.1:9999
https://medicine.example.com/flask/   -> Flask YOLO 127.0.0.1:5000
https://medicine.example.com/socket.io/ -> Flask Socket.IO
```

App 内所有接口都使用云服务器入口，不再使用 `127.0.0.1`、`localhost`、电脑局域网 IP。当前临时使用 `http://47.118.23.173`，正式发布建议换成 HTTPS 域名。

## 5. 前端必须改造的地址

当前 Vue 代码里存在本地地址，打包成手机 App 后必须替换。

需要处理的典型位置：

```text
src/utils/request.ts
src/utils/socket.ts
src/views/imgPredict/index.vue
src/views/videoPredict/index.vue
src/views/cameraPredict/index.vue
src/views/imgPredictBatch/index.vue
src/views/personal/index.vue
src/views/userManage/dialog.vue
src/views/videoRecord/index.vue
vite.config.ts
.env
.env.development
.env.production
```

当前问题：

- `src/utils/request.ts` 读取的是 `import.meta.env.VITE_API_DOMAIN`。
- `.env.development` 和 `.env.production` 里现在配置的是 `VITE_API_URL`，变量名不一致。
- 多个上传组件写死 `http://127.0.0.1:9999/files/upload`。
- 视频和摄像头检测写死 `http://127.0.0.1:5000/predictVideo`、`http://127.0.0.1:5000/predictCamera`。
- Socket 写死 `http://127.0.0.1:5000`。
- `window.open('http://localhost:8888/#/videoShow?id=' + row.id)` 不适合手机 App。

建议统一新增环境变量：

```env
VITE_API_DOMAIN=https://medicine.example.com
VITE_SPRING_API=https://medicine.example.com/api
VITE_FLASK_API=https://medicine.example.com/flask
VITE_SOCKET_URL=https://medicine.example.com
```

开发环境可以临时写成：

```env
VITE_API_DOMAIN=http://服务器公网IP
VITE_SPRING_API=http://服务器公网IP/api
VITE_FLASK_API=http://服务器公网IP/flask
VITE_SOCKET_URL=http://服务器公网IP
```

正式 App 建议必须改成 HTTPS。

## 6. 前端接口调用规范

Spring Boot 接口统一使用：

```ts
const springApi = import.meta.env.VITE_SPRING_API;
```

Flask 检测接口统一使用：

```ts
const flaskApi = import.meta.env.VITE_FLASK_API;
```

上传地址统一使用：

```ts
const uploadUrl = `${import.meta.env.VITE_SPRING_API}/files/upload`;
```

Socket 地址统一使用：

```ts
io(import.meta.env.VITE_SOCKET_URL, {
  path: '/socket.io',
  transports: ['websocket', 'polling'],
});
```

不要在业务页面里再写死 IP、端口或 localhost。

## 7. Android App 工程创建流程

在 `MedicineDetection_vue` 目录执行：

```bash
npm install
npm install @capacitor/core @capacitor/cli @capacitor/android
npm run build
npx cap init MedicineDetection com.example.medicinedetection --web-dir=dist
npx cap add android
npx cap copy android
npx cap open android
```

后续每次修改 Vue 后：

```bash
npm run build
npx cap copy android
```

如果安装了新的 Capacitor 插件：

```bash
npx cap sync android
```

## 8. Android 配置要点

如果接口已经是 HTTPS，一般不需要允许明文 HTTP。

如果毕业设计临时使用 HTTP，需要在 Android 配置中允许明文请求：

```xml
android:usesCleartextTraffic="true"
```

正式版本建议：

- 使用 HTTPS 域名。
- 不在 App 中写死公网 IP。
- 不把数据库密码、OpenAI Key、服务器密码写进前端代码。

## 9. 手机端测试流程

1. 确认阿里云服务器安全组开放 80、443。
2. 确认 Spring Boot、Flask、MySQL 在服务器正常运行。
3. 用手机浏览器访问：

```text
https://medicine.example.com/api/user/all
https://medicine.example.com/flask/predictVideo
```

具体接口可能需要参数，能返回后端响应或业务错误即可，不能是连接失败。

4. 构建 Vue 并同步 Android。
5. Android Studio 连接手机运行。
6. 测试登录、上传图片、图片检测、视频检测、历史记录。

## 10. 开发里程碑

第一阶段：云端可用

- Spring Boot 能连接云端 MySQL。
- Flask 能加载 YOLO 权重。
- Nginx 能通过 `/api` 和 `/flask` 转发。
- 文件上传和访问路径正常。

第二阶段：前端适配

- 所有本地接口地址改为环境变量。
- 上传组件使用云端上传地址。
- Socket 使用云端地址。
- 手机浏览器访问前端功能正常。

第三阶段：App 打包

- Capacitor Android 工程生成成功。
- 真机运行 App。
- 图片、视频、历史记录主流程通过。

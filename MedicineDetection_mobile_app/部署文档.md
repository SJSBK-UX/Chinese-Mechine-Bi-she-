# 药品检测系统云服务器部署文档

## 1. 部署目标

将药品检测系统部署到 4 核 8G 阿里云服务器，并为 Android App 提供稳定的 HTTPS 接口。

部署后的服务结构：

```text
Nginx 443
  /api/       -> Spring Boot 9999
  /flask/     -> Flask YOLO 5000
  /socket.io/ -> Flask Socket.IO 5000

Spring Boot -> MySQL yolo
Flask YOLO  -> Spring Boot 文件上传接口
```

## 2. 服务器建议配置

硬件：

```text
CPU: 4 核
内存: 8 GB
系统盘: 建议 80 GB 以上
带宽: 建议 3 Mbps 以上，视频演示最好 5 Mbps 以上
```

系统建议：

```text
Ubuntu 22.04 LTS
```

如果你的阿里云系统是 Alibaba Cloud Linux 或 CentOS，安装命令需要把 `apt` 替换为 `yum` 或 `dnf`。

## 3. 阿里云安全组

安全组入方向建议：

```text
22/tcp    SSH 管理，建议只允许自己的 IP
80/tcp    HTTP，用于证书申请和跳转 HTTPS
443/tcp   HTTPS，App 正式访问入口
```

不要对公网开放：

```text
3306 MySQL
5000 Flask
9999 Spring Boot
```

## 7. MySQL 部署

登录 MySQL：

```bash
sudo mysql
```

创建数据库和账号：

```sql
CREATE DATABASE yolo DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
CREATE USER 'yolo_user'@'localhost' IDENTIFIED BY '替换为强密码';
GRANT ALL PRIVILEGES ON yolo.* TO 'yolo_user'@'localhost';
FLUSH PRIVILEGES;
```

导入数据：

```bash
mysql -u yolo_user -p yolo < /opt/medicine-detection/app/yoloai.sql
```

确认数据库：

```bash
mysql -u yolo_user -p -e "SHOW TABLES FROM yolo;"
```

## 8. Spring Boot 配置与启动

进入后端目录：

```bash
cd /opt/medicine-detection/app/MedicineDetection_springboot
```

建议将 `application.properties` 调整为云端配置：

```properties
spring.application.name=MedicineDetection
mybatis.mapper-locations=classpath:mappers/*xml
mybatis.type-aliases-package=com.example.medicinedetection.entity
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.datasource.url=jdbc:mysql://localhost:3306/yolo?serverTimezone=Asia/Shanghai&useUnicode=true&characterEncoding=utf8&useSSL=false
spring.datasource.username=yolo_user
spring.datasource.password=替换为强密码
server.port=9999
server.address=127.0.0.1
file.ip=medicine.example.com
spring.servlet.multipart.max-file-size=500MB
spring.servlet.multipart.max-request-size=500MB
```

如果代码里的文件访问 URL 依赖 `file.ip` 和 `server.port` 拼接，建议后续改成完整外部地址，例如：

```properties
file.base-url=https://medicine.example.com/api/files
file.storage-dir=/opt/medicine-detection/data/files
```

构建：

```bash
mvn -DskipTests package
```

临时启动测试：

```bash
java -jar target/*.jar
```

创建 systemd 服务：

```bash
sudo tee /etc/systemd/system/medicine-spring.service >/dev/null <<'EOF'
[Unit]
Description=Medicine Detection Spring Boot Service
After=network.target mysql.service

[Service]
Type=simple
WorkingDirectory=/opt/medicine-detection/app/MedicineDetection_springboot
ExecStart=/usr/bin/java -jar /opt/medicine-detection/app/MedicineDetection_springboot/target/medicine-detection-0.0.1-SNAPSHOT.jar
Restart=always
RestartSec=5
User=你的Linux用户名
Environment=TZ=Asia/Shanghai

[Install]
WantedBy=multi-user.target
EOF
```

如果实际 jar 文件名不同，需要把 `ExecStart` 里的 jar 名改成真实文件名。

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now medicine-spring
sudo systemctl status medicine-spring
```

查看日志：

```bash
journalctl -u medicine-spring -f
```

## 9. Flask YOLO 配置与启动

进入 Flask 目录：

```bash
cd /opt/medicine-detection/app/MedicineDetection_flask
```

创建虚拟环境：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
```

安装依赖。根据当前代码至少需要：

```bash
pip install flask flask-socketio ultralytics opencv-python requests openai
```

服务器无图形界面时，如果 `opencv-python` 报错，改用：

```bash
pip uninstall -y opencv-python
pip install opencv-python-headless
```

确认权重文件存在：

```text
/opt/medicine-detection/app/MedicineDetection_flask/weights/best.pt
```

当前 `utils/Fun.py` 默认上传到：

```text
http://127.0.0.1:9999/files/upload
```

服务器上这样是可用的，因为 Flask 和 Spring Boot 都在同一台服务器。但更规范的方式是后续改成环境变量，例如：

```text
SPRING_UPLOAD_URL=http://127.0.0.1:9999/files/upload
```

临时启动测试：

```bash
python main.py
```

创建 systemd 服务：

```bash
sudo tee /etc/systemd/system/medicine-flask.service >/dev/null <<'EOF'
[Unit]
Description=Medicine Detection Flask YOLO Service
After=network.target medicine-spring.service

[Service]
Type=simple
WorkingDirectory=/opt/medicine-detection/app/MedicineDetection_flask
ExecStart=/opt/medicine-detection/app/MedicineDetection_flask/.venv/bin/python /opt/medicine-detection/app/MedicineDetection_flask/main.py
Restart=always
RestartSec=5
User=你的Linux用户名
Environment=TZ=Asia/Shanghai
Environment=FLASK_HOST=127.0.0.1
Environment=FLASK_PORT=5000

[Install]
WantedBy=multi-user.target
EOF
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now medicine-flask
sudo systemctl status medicine-flask
```

查看日志：

```bash
journalctl -u medicine-flask -f
```

## 10. Nginx 配置

创建站点配置：

```bash
sudo tee /etc/nginx/sites-available/medicine-detection >/dev/null <<'EOF'
server {
    listen 80;
    server_name medicine.example.com;

    client_max_body_size 500m;

    location /api/ {
        proxy_pass http://127.0.0.1:9999/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /flask/ {
        proxy_pass http://127.0.0.1:5000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 600s;
        proxy_send_timeout 600s;
    }

    location /socket.io/ {
        proxy_pass http://127.0.0.1:5000/socket.io/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 600s;
    }
}
EOF
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/medicine-detection /etc/nginx/sites-enabled/medicine-detection
sudo nginx -t
sudo systemctl reload nginx
```

如果系统没有 `sites-available`，可以把配置放到：

```text
/etc/nginx/conf.d/medicine-detection.conf
```

## 15. 常见问题

### 15.1 App 提示网络连接错误

检查：

- App 里是否还有 `127.0.0.1` 或 `localhost`。
- 域名是否解析到阿里云公网 IP。
- 安全组是否开放 443。
- Nginx 是否运行。
- HTTPS 证书是否有效。

### 15.2 上传失败

检查：

- Nginx `client_max_body_size 500m`。
- Spring Boot `spring.servlet.multipart.max-file-size=500MB`。
- Spring Boot 文件目录是否有写入权限。

### 15.3 Flask 检测慢

4 核 8G CPU 推理视频会慢，属于正常现象。答辩演示建议：

- 使用短视频。
- 优先演示图片检测。
- 降低视频分辨率。
- 使用较小 YOLO 权重。

### 15.4 Socket 连接失败

检查 Nginx 是否配置：

```nginx
proxy_http_version 1.1;
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection "upgrade";
```

同时确认前端 Socket 地址使用：

```text
https://medicine.example.com
```

而不是：

```text
http://127.0.0.1:5000
```

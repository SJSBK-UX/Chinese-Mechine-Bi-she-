# 基于 YOLO11 和 DeepSeek 的中药材检测与识别系统项目说明书

## 一、项目概述

本项目是一个面向中药材图像检测与识别场景的毕业设计系统，主要实现图片识别、批量图片识别、视频识别、摄像头识别、AI 文字说明、历史记录管理和手机端访问等功能。系统采用前后端分离架构，前端负责交互展示，Spring Boot 后端负责业务接口和数据存储，Flask 后端负责 YOLO11 模型推理和 DeepSeek 文本说明生成，Android 端通过 WebView 实现移动端访问。

项目核心思路可以概括为：用户上传图片、视频或摄像头画面后，系统将数据传递到后端，由 YOLO11 模型完成中药材目标检测，返回类别、置信度和检测框；随后结合 DeepSeek 生成文字说明，并将识别记录保存到 MySQL 数据库中，最终在网页端或手机端展示识别结果。

## 二、系统功能

| 功能模块 | 功能说明 |
| --- | --- |
| 用户管理 | 支持用户注册、登录和个人信息维护 |
| 单图识别 | 上传单张中药材图片，返回识别类别、置信度和检测框 |
| 批量识别 | 支持多张图片批量上传，并展示批量检测结果 |
| 视频识别 | 对上传视频进行逐帧检测，生成带检测框的结果视频 |
| 摄像头识别 | 调用摄像头画面进行实时检测，并动态显示识别结果 |
| AI 文字说明 | 结合 DeepSeek 根据识别结果生成自然语言说明 |
| 历史记录 | 保存图片、视频和摄像头识别记录，支持查询与删除 |
| 手机端访问 | Android WebView 加载网页系统，实现移动端访问 |

## 三、技术架构

系统整体采用“Vue 前端 + Spring Boot 业务后端 + Flask AI 推理后端 + MySQL 数据库 + Android WebView”的架构。

```text
用户图片/视频/摄像头画面
        |
        v
Vue3 前端页面
        |
        v
Spring Boot 业务后端
        |
        +----> MySQL 数据库保存识别记录
        |
        v
Flask AI 推理服务
        |
        +----> YOLO11 模型检测
        |
        +----> DeepSeek 生成文字说明
        |
        v
返回类别、置信度、检测框和 AI 建议
```

各部分职责如下：

| 层次 | 技术 | 主要职责 |
| --- | --- | --- |
| 前端展示层 | Vue3、Element Plus、Vite | 页面展示、图片上传、结果渲染、历史记录查询 |
| 业务后端层 | Spring Boot、MyBatis Plus | 用户管理、文件上传、接口转发、识别记录保存 |
| AI 推理层 | Flask、Ultralytics YOLO、DeepSeek API | 图片/视频/摄像头推理、AI 文本说明 |
| 数据存储层 | MySQL | 保存用户信息和识别历史 |
| 移动访问层 | Android WebView | 在手机端加载网页系统 |

```text
    |

    |

```

## 十三、使用注意事项

1. 启动系统前，需要先确认 MySQL 已启动，并且 `yolo` 数据库已导入。
2. 如果 `9999`、`5000` 或 `8888` 端口被占用，需要先关闭占用端口的程序，或者修改对应配置。
3. 视频识别会比图片识别耗时更长，CPU 环境下等待时间可能较明显。
4. DeepSeek 文字说明依赖网络和 API 配置，如果接口不可用，图片检测功能仍可运行，但 AI 建议可能无法生成。
5. 手机端 WebView 加载的是网页系统地址，如果部署地址发生变化，需要同步修改 Android 端加载地址。
6. 如果模型识别效果不稳定，可以适当调整置信度阈值，或补充更多训练数据重新训练模型。

## 十四、常见问题

### 1. 前端页面打不开怎么办？

先确认 Vue 服务是否启动成功，并查看终端是否显示 `http://localhost:8888`。如果端口被占用，可以修改 `MedicineDetection_vue/.env` 中的 `VITE_PORT`。

### 2. 图片上传成功但识别失败怎么办？

需要检查 Spring Boot 和 Flask 是否都已启动。Spring Boot 默认端口为 `9999`，Flask 默认端口为 `5000`。同时确认模型文件 `best.pt` 是否存在。

### 3. 数据库连接失败怎么办？

检查 MySQL 是否启动、数据库名是否为 `yolo`，以及 `application.properties` 中的用户名和密码是否正确。

### 4. DeepSeek 没有返回说明怎么办？

检查网络连接、API Key 配置和 DeepSeek 接口是否可用。DeepSeek 只负责文字说明，不影响 YOLO11 的图像检测结果。

### 5. 视频识别速度慢怎么办？

视频识别需要对视频帧逐帧推理，CPU 环境下速度较慢是正常情况。后续可以通过 GPU 推理、ONNX Runtime、降低视频分辨率或抽帧检测来优化。
